## อัปเดต Plesk 500 / `.htaccess` 2026-06-30

หลัง deploy แล้วเว็บจริง `new.subzerowolf-sea.com` เคยขึ้น `Internal Server Error` บน route เช่น:

- `/refrigeration/discover-sub-zero`
- `/products`
- `/showroom`
- `/journal`

ตรวจ log ใน Plesk พบ error:

`AH00124: Request exceeded the limit of 10 internal redirects due to probable configuration error`

สาเหตุคือ `.htaccess` เดิมทำ Apache internal redirect loop ไม่ใช่โค้ด Next.js พัง

วิธีแก้ที่ทำแล้ว:

- เปลี่ยนชื่อ `.htaccess` บน server เป็น `.htaccess.bak`
- Restart App ใน Plesk
- ตรวจเว็บจริงแล้ว route หลักกลับมา `200`
- เพิ่มไฟล์ `.htaccess` แบบปลอดภัยใน repo โดยมีเฉพาะ comment และไม่มี rewrite rule
- เพิ่ม `.htaccess.bak` ใน `.gitignore`
- เพิ่ม Plesk deploy notes ใน `README.md`

ข้อควรจำ:

- ห้ามเอา `.htaccess.bak` กลับมาเป็น `.htaccess` ถ้ายังไม่ได้ตรวจ rule
- Plesk Node.js / Passenger ใช้ `app.js` เป็น startup file และ Next.js handle route เอง
- หลัง pull/deploy ให้รัน `npm install`, `npm run build`, แล้วกด `Restart App`
